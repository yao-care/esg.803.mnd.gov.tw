# PLN 類型規範 / PLN Type Specification

## 必要章節結構

計畫文件（Plan）須依以下順序呈現 6 個章節，所有章節均為必要：

1. **【必要】計畫目的與範圍** — 說明本計畫的目的、適用對象、計畫期間
2. **【必要】目標與 KPI** — 列出量化目標及對應 KPI，每個 KPI 含基準值、目標值、衡量方法
3. **【必要】行動方案表格** — Markdown 表格，含行動項目、負責人、期限、預算（若有）、狀態欄位
4. **【必要】資源需求** — 說明執行計畫所需的人力、預算、工具、外部資源
5. **【必要】時程表** — 以甘特圖（Mermaid gantt）或里程碑表格呈現整體時程
6. **【必要】監控與檢討機制** — 說明進度監控頻率、回報對象、定期檢討會議安排

## 語氣與用詞規範

- 行動方案表格中，負責人欄位填寫職稱或角色名稱（非個人姓名）
- 期限欄位格式統一為 YYYY-MM-DD 或 YYYY-QN（季度）
- 狀態欄位用詞統一：「未開始」「進行中」「已完成」「延遲」

## 必要元素

- **【必要】行動方案表格**：Markdown 表格，含行動項目、負責人、期限、狀態四欄（最少）
- **【必要】負責人欄位**：每個行動項目均明確指定負責人（職稱或角色）
- **【必要】期限欄位**：每個行動項目均含具體期限（YYYY-MM-DD 格式或季度）

## 交叉引用規則

- 應引用相關 POL 政策文件作為計畫依據
- 應引用相關 PRO 程序文件說明執行方式
- 可引用 MTX 矩陣作為現況分析依據

## 審查重點

- **PL1 行動表格完整**：行動方案表格含所有必要欄位，無空白行
- **PL2 負責人明確**：每行均有具體負責人（職稱或角色），禁止「待定」「TBD」
- **PL3 期限具體**：每行均有具體期限，禁止「盡快」「適時」等模糊用詞

## 類型特定關鍵字

`行動表格`、`負責人`、`期限`
