# MTX 類型規範 / MTX Type Specification

## 必要章節結構

矩陣文件（Matrix）須依以下順序呈現 5 個章節，所有章節均為必要：

1. **【必要】矩陣目的與範圍** — 說明本矩陣的用途、涵蓋範圍、使用情境
2. **【必要】列/欄定義** — 清楚說明矩陣的行（row）代表什麼、列（column）代表什麼，及各儲存格的含義
3. **【必要】資料來源** — 列出矩陣數據取自哪些文件（document_id），說明數據擷取方式
4. **【必要】更新頻率** — 說明矩陣應於何時更新（觸發條件或週期）、由誰負責更新
5. **【必要】追溯性** — 說明矩陣中每筆數據如何追溯至來源文件或原始記錄

## 語氣與用詞規範

- 矩陣本體使用 Markdown 表格呈現
- 儲存格內容應簡潔（建議 30 字以內），詳細說明放入備註欄或附錄
- 來源引用格式：`[文件名稱](../document_id/)` 或直接列出 document_id

## 必要元素

- **【必要】表格格式**：矩陣本體以 Markdown 表格呈現，有表頭（header row）
- **【必要】資料來源引用**：資料來源章節明確列出來源文件的 document_id
- **【必要】更新頻率**：明確說明更新週期（如「每季」「每次 RPT 發行後」）或觸發條件

## 交叉引用規則

- **必須引用**來源文件（RPT、FRM 或 STD 的 document_id）
- 可被 RPT 引用（排放源清冊矩陣）
- 可被 PRO 引用（流程對照矩陣）

## 審查重點

- **M1 表格格式**：矩陣本體為有效的 Markdown 表格，含表頭列
- **M2 來源追溯性**：資料來源章節列出至少一份來源文件的 document_id，每筆數據可追溯

## 類型特定關鍵字

`矩陣`、`對照`、`追溯`
