# REG 類型規範 / REG Type Specification

## 必要章節結構

紀錄文件（Record）須依以下順序呈現 5 個章節，所有章節均為必要：

1. **【必要】紀錄目的與範圍** — 說明本紀錄的用途、記錄對象、適用範圍
2. **【必要】紀錄格式** — 以 Markdown 表格定義所有欄位（欄位名稱、資料型態、說明、必填/選填）
3. **【必要】填寫規範** — 說明由誰填寫、何時填寫、如何填寫；含填寫頻率及責任人
4. **【必要】更新頻率** — 明確聲明更新週期（如「每月」「每季」「每次排放活動後」）或觸發更新的條件
5. **【必要】保存與調閱** — 說明保存期限、儲存位置、調閱權限、銷毀程序

## 語氣與用詞規範

- 填寫規範使用祈使句（如「每月底前更新」「由設施管理員填入」）
- 更新頻率須明確（禁止「定期」「必要時」等模糊用詞）
- 保存期限須以年數或具體期間表示（如「保存 7 年」「至合約終止後 3 年」）

## 必要元素

- **【必要】表格格式**：紀錄格式章節以 Markdown 表格定義所有欄位
- **【必要】更新頻率聲明**：更新頻率章節明確說明更新週期或觸發條件

## 交叉引用規則

- 應引用上位 PRO 程序文件（說明本紀錄在程序中的角色）
- 可被 RPT 引用（作為排放數據的原始紀錄來源）
- 可被 MTX 引用（作為矩陣的數據來源）

## 審查重點

- **RE1 表格格式定義**：紀錄格式章節含有效的 Markdown 表格，所有欄位均有說明
- **RE2 更新頻率**：更新頻率章節含具體週期或觸發條件，無模糊用詞

## 類型特定關鍵字

`表格格式`、`更新頻率`
