# GDL 類型規範 / GDL Type Specification

## 必要章節結構

指引文件（Guideline）須依以下順序呈現 5 個章節，所有章節均為必要：

1. **【必要】指引目的與範圍** — 說明本指引的目的、適用對象、與強制性規範的差異（非強制性）
2. **【必要】適用對象** — 明確列出本指引適用的人員職稱/角色、設施類型或業務情境
3. **【必要】指引內容** — 含公式（以 LaTeX 或說明文字呈現）、計算範例、逐步操作說明
4. **【必要】參考文獻** — 列出引用的來源文件（含 document_id 及外部標準名稱）
5. **【必要】附錄** — 完整計算範例、對照表、常見問題（Q&A）

## 語氣與用詞規範

- 指引文件為非強制性，使用 should（宜）或 may（得）語氣，避免 shall（應）
- 指引內容使用說明性語句（如「建議使用」「可參考以下公式」）
- 公式中的變數須有定義說明表（變數名稱、單位、說明）

## 必要元素

- **【必要】公式或計算方法**：指引內容章節含至少一個完整公式或計算方法說明
- **【必要】範例**：含具體數字的計算範例，展示公式的實際應用
- **【必要】參考文獻**：列出引用來源，含 document_id（內部文件）或標準名稱（外部來源）

## 交叉引用規則

- 應引用相關 RPT 報告文件（說明本指引如何支援報告編製）
- 應引用相關 PRO 程序文件（說明本指引在程序中的使用時機）
- 可引用外部標準（STD）作為計算方法的依據來源

## 審查重點

- **G1 公式完整**：指引內容章節含完整公式，且公式中所有變數均有定義
- **G2 範例提供**：附錄或指引內容章節含具體數字的計算範例
- **G3 參考引用**：參考文獻章節列出至少一份來源（內部 document_id 或外部標準名稱）

## 類型特定關鍵字

`參考`、`公式`、`範例`
