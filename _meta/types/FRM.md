# FRM 類型規範 / FRM Type Specification

## 必要章節結構

表單文件（Form）須依以下順序呈現 5 個章節，所有章節均為必要：

1. **【必要】表單資訊（編號/版本/分類）** — document_id（格式 FRM-NNN）、版本號、對應程序 PRO document_id、保存期限
2. **【必要】填寫說明** — 祈使句，逐一說明每個欄位的填寫方式；必填欄位標示 *
3. **【必要】表單欄位（含必填標示）** — Markdown 表格，欄位含：欄位名稱、說明、格式/範例、必填/選填
4. **【必要】核准與簽核區段** — 核准者職位/角色、姓名欄、日期欄；至少一個核准層級
5. **【必要】附件與備註** — 說明可附加的佐證文件類型及備註空白區

## 語氣與用詞規範

- 填寫說明使用祈使句（如「填寫本月排放量」「選擇適用類別」「附上計算明細」）
- 必填欄位在欄位名稱後標示 * 符號
- 核准區段用詞：「核准者」「審查者」「填表人」（職稱或角色，非姓名）

## 必要元素

- **【必要】必填欄位標示 \***：表單欄位表格中，每個必填欄位的欄位名稱後加 *
- **【必要】核准/簽核區段（含日期欄）**：核准章節含核准者職稱欄、姓名欄、日期欄
- **【必要】填寫說明（每個欄位的填法）**：每個欄位均有對應的填寫說明
- **【必要】文件 frontmatter**：須含 document_id（格式 FRM-NNN）

## 交叉引用規則

- **必須引用**上位 PRO 程序文件（於表單資訊章節列出 parent PRO document_id）
- FRM 為 PRO 附錄的組成部分，須能從 PRO 追溯至 FRM

## 審查重點

- **F1 必填標示 \***：表單欄位表格所有必填欄位均標示 *，且填寫說明中有對應說明
- **F2 核准區段**：核准與簽核章節含核准者職位/角色欄、姓名欄、日期欄三欄

## 類型特定關鍵字

`必填`、`核准區段`、`填寫說明`
