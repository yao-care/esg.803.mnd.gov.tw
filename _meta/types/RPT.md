# RPT 類型規範 / RPT Type Specification

## 必要章節結構

報告文件（Report）須依以下順序呈現 7 個章節，所有章節均為必要：

1. **【必要】目的與適用範圍** — 說明本報告的目的、適用標準（如 ISO 14064-1）、報告期間
2. **【必要】組織邊界** — 採用營運控制法界定，逐一列出納入設施與排除設施，說明排除理由
3. **【必要】報告邊界** — 鑑別類別 1（直接排放）與類別 2（能源間接排放）之排放源，說明鑑別依據
4. **【必要】排放數據** — 呈現各類別排放量彙總表（含百分比）；表格格式為 Markdown 表格
5. **【必要】數據品質管理** — 說明計算方法、排放係數來源、不確定性分析結果
6. **【必要】查證** — 呈現內部查證及外部查證結果與查證聲明
7. **【必要】報告管理** — 版本控制紀錄、發行日期、核准者簽署

## 語氣與用詞規範

- 須使用 ISO 14064-1 術語（如「直接排放」「能源間接排放」「活動數據」「排放係數」「不確定性」）
- 數據呈現須含單位（公噸 CO₂e）
- 查證聲明須明確說明合理保證或有限保證等級

## 必要元素

- **【必要】數據年度**：明確標示報告所涵蓋的會計年度
- **【必要】基準年引用**：說明基準年設定及重新計算政策
- **【必要】總排放量數字**：以公噸 CO₂e 呈現，類別 1 與類別 2 分項列出
- **【必要】類別 1/2 分項**：排放數據章節含類別 1、類別 2 各自的數值與佔比

## 交叉引用規則

- **必須連結** MTX（排放源清冊矩陣）：於報告邊界或排放數據章節引用對應 MTX document_id
- 可引用 STD（排放係數標準）作為計算依據來源

## 審查重點

- **R1 邊界完整性**：組織邊界與報告邊界章節列出所有納入/排除設施，且說明排除理由
- **R2 排放數據一致性**：總排放量數字與各類別分項加總一致
- **R3 不確定性分析**：數據品質管理章節含不確定性分析結果（百分比或等級）

## 類型特定關鍵字

`數據年度`、`總排放量`、`邊界`
