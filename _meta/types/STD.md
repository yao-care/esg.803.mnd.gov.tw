# STD 類型規範 / STD Type Specification

## 必要章節結構

標準文件（Standard）須依以下順序呈現 5 個章節，所有章節均為必要：

1. **【必要】標準概述** — 引用來源（外部標準或法規名稱）、版本號、發布機構、生效日期
2. **【必要】適用條款摘錄** — 逐條列出與本院業務相關的條款，每條款含條款號及原文或中譯
3. **【必要】合規要求** — 列出本院應符合的具體要求，每項要求可追溯至來源條款號
4. **【必要】合規對照** — 以表格呈現條款號、合規要求、本院現行做法、符合狀態（符合/部分符合/不符合）
5. **【必要】更新追蹤** — 說明外部標準修訂時的更新機制，含負責人、更新週期、通知流程

## 語氣與用詞規範

- 摘錄外部條款時，保留原條款號碼格式（如「第 X 條」「Article X」「Section X.X」）
- 合規要求使用「應」或 shall 語氣
- 合規對照表中，符合狀態用詞統一：「符合」「部分符合」「不符合」「不適用」

## 必要元素

- **【必要】來源歸屬**：標準概述章節明確列出法規名稱、條款號、發布機構
- **【必要】合規要求清單**：每項要求可追溯至來源條款號
- **【必要】合規對照表**：Markdown 表格，含條款號、要求、現行做法、符合狀態四欄

## 交叉引用規則

- **必須引用**外部標準來源（法規名稱、發布機構、版本）
- 可被 RPT、PRO、POL 引用作為規範依據

## 審查重點

- **ST1 來源歸屬**：標準概述明確列出發布機構與版本，禁止「詳見官網」等模糊說法
- **ST2 條款引用**：適用條款摘錄每條均含條款號
- **ST3 合規對照**：合規對照章節以表格呈現，每行含符合狀態

## 類型特定關鍵字

`條款`、`要求`、`合規`
