## 章節標題格式

### 儲存格式（Markdown）

章節標題使用數字編號 + Markdown heading：

```
## 1. 目的
## 2. 適用範圍
## 3. 權責
## 4. 作業流程
```

子章節使用「X.Y」格式：

```
### 4.1 申請階段
### 4.2 審核階段
```

### 輸出格式（HTML）

build 時自動將數字編號轉換為中文編號：

- `## 1. ` → `一、`
- `## 2. ` → `二、`
- `### 4.1 ` → `（一）`

此轉換由 build.js 的 renderMarkdownToHtml 處理，Markdown 原始檔一律使用數字格式。

### 禁止

- 禁止在 Markdown 中直接使用「一、」「二、」中文編號作為 heading
- 禁止混用數字和中文編號

## 引文格式

### 儲存格式（Markdown 原始檔內的交叉引用）

引用其他文件時使用文件編號：

- 正確：「依本局 ISMS-203 辦理」
- 正確：「參照 PIMS-301 第 4 節」

### AI 回答格式（system prompt 規範）

AI 回答中的引文使用以下格式：

[來源:文件編號#章節]

規則：
- 冒號使用半形 `:`（U+003A），不使用全形 `：`（U+FF1A）
- 文件編號使用實際編號（如 ISMS-303、PIMS-301）
- 章節使用數字格式（如 #2. 作業流程）
- 一個回答可包含多個引文

### 防護機制

即使 AI 未遵循上述格式，assistant.html 的 autoLinkDocKeys 機制會自動偵測回答中出現的所有已知文件編號並建立連結。此機制不依賴 AI 的格式遵循度。
